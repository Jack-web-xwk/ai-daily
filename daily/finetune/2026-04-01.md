## 🔥 要闻（Top 5条）

**1. ERPO：针对大推理模型的 token 级熵正则化策略优化** → 提出 ERPO 替代标准 GRPO，通过识别"关键决策枢纽点"(CDPs) 在 token 级别进行自适应探索增强，解决了均匀优势信号导致的熵坍缩和冗余推理路径问题。在 MATH、AIME 等数学推理基准上显著优于 GRPO，同时生成更简洁的推理链。 → 🔗 [arXiv:2603.28204](https://arxiv.org/abs/2603.28204)

**2. HISA：为 DeepSeek-V3.2 设计的层级稀疏注意力索引** → 提出 HISA 作为 DeepSeek Sparse Attention 索引器的即插即用替代方案，将平坦 token 扫描转化为两级层级搜索（块级粗筛 + token 级精筛），在 32K 上下文下实现 2× 加速、128K 下下文下 4× 加速，无需额外训练。 → 🔗 [arXiv:2603.28458](https://arxiv.org/abs/2603.28458)

**3. IsoQuant：基于四元数的 KV Cache 量化压缩** → 提出基于四元数代数和 SO(4) 等角分解的分块旋转框架，用于 LLM KV Cache 的低比特在线向量量化。在 d=128 下将旋转计算量从 ~2408 FMA 降至 512，CUDA 内核级加速达 4.5×-6×。 → 🔗 [arXiv:2603.28430](https://arxiv.org/abs/2603.28430)

**4. 用 LLM 进化发现强化学习算法** → GECCO 2026 录用论文。提出基于进化的框架，使用 LLM 作为变异算子直接在可执行更新规则上搜索新 RL 算法，排除 Actor-Critic、TD 等标准机制以促进创新，发现的算法在多个 Gymnasium 基准上与 SAC/PPO/DQN 竞争力相当。 → 🔗 [arXiv:2603.28416](https://arxiv.org/abs/2603.28416)

**5. Unsloth Studio 更新：多 GPU 微调 + 工具调用大幅改进** → Unsloth 于3月31日发布 Studio 重大更新：支持多 GPU 自动分配进行微调和推理，工具调用准确率提升 30%-80%，支持自定义 GGUF 文件夹，修复 Windows 兼容性问题。 → 🔗 [GitHub Releases](https://github.com/unslothai/unsloth/releases)

## 🔬 论文速递

**📐 优化器方向**

- **Balancing Before Orthogonalization：Muon 优化器的轻量级均衡化** → 提出在正交化前对动量矩阵进行行/列归一化的预处理方法，仅需 O(m+n) 额外状态。在 LLaMA2 (130M/350M) C4 预训练上一致优于标准 Muon，收敛更快、验证困惑度更低。 → 🔗 [arXiv:2603.28254](https://arxiv.org/abs/2603.28254)

**🤖 自适应与微调**

- **PACE：无反向传播的持续测试时自适应** → 提出基于 CMA-ES 进化策略的无梯度自适应系统，直接优化归一化层的仿射参数，在低维子空间中搜索。相比现有无反向传播方法运行时减少 50%+，达到 SOTA 准确率。 → 🔗 [arXiv:2603.28678](https://arxiv.org/abs/2603.28678)

- **LIBERO-Para：VLA 模型微调的释义鲁棒性基准** → 发现 7 个 VLA 配置（0.6B-7.5B）在释义指令下性能下降 22-52 个百分点，80-96% 失败来自规划层轨迹偏移而非执行错误。提出 PRIDE 指标量化释义难度。 → 🔗 [arXiv:2603.28301](https://arxiv.org/abs/2603.28301)

**🧠 表征与可解释性**

- **SAE 为何在组合泛化上失败** → 证明稀疏自编码器(SAE)的失败源于字典学习而非摊销推断：SAE 学到的字典方向系统性偏差，用 FISTA 替换编码器也无法弥补，Oracle 字典在所有测试规模上均证明问题可解。 → 🔗 [arXiv:2603.28744](https://arxiv.org/abs/2603.28744)

- **XAI 本质是因果推断** → 立场论文论证可解释 AI 的持续争议源于缺少因果模型这一"隐藏的地面真值"，将 XAI 查询统一框架为因果推断问题。 → 🔗 [arXiv:2603.28597](https://arxiv.org/abs/2603.28597)

## 💡 今日洞察

今天最值得关注的趋势是 **RL 对齐技术的 token 级精细化**。ERPO 论文精准击中了 GRPO 的核心缺陷——粗粒度序列级优势分配导致熵坍缩，而"关键决策枢纽点"(CDPs) 的发现为 RLHF/DPO/GRPO 类方法提供了新的优化粒度。这与 torchtune v0.6.0 正式支持分布式 DPO 形成呼应，说明产业界和学术界都在加速推进对齐技术的工程化和精细化。

另一个信号是 **效率优化的全面开花**：HISA 针对注意力、IsoQuant 针对 KV Cache、Muon 均衡化针对优化器、PACE 针对测试时自适应——每个环节都在被精打细算地压缩。微调的终极形态可能不是更大的 GPU 集群，而是每一层计算都被极致优化后的"刚好够用"。
