## 🎯 今日可直接落地

### 1. vLLM v0.18.0 升级：推理性能跃迁
**能做什么：** 部署或升级 vLLM 到 v0.18.0，启用以下新能力：
- **gRPC 服务**：用 `--grpc` flag 替代 HTTP，微服务集成更原生
- **智能 KV Cache CPU 卸载**：自动保留高频 block，冷数据卸载到内存
- **NGram GPU 推测解码**：spec decode 迁移 GPU，延迟降低

**预期效果：** 吞吐提升 15-30%（取决于 batch size 和模型），长上下文场景 KV Cache 内存节省 40%+

**落地建议：** 先在测试环境用 Qwen3.5 跑基准，对比 v0.17.x 的 TPS 和首 token 延迟

### 2. LoRA 顺序微调 + Agent 优化器：低成本多任务适配
**能做什么：** 用"Greedy Is a Strong Default"论文的思路，用 Agent 迭代优化替代 Optuna 做 LoRA 超参搜索

**落地场景：**
- 客服机器人：先微调意图识别 → 再微调情绪分析 → 再微调知识问答，顺序叠加不遗忘
- 垂直行业：医疗/法律领域依次适配不同子任务

**预期效果：** SST-2 上从 89.5% → 92.7%，超参搜索效率翻倍

**工具链：** LoRA + Agent 优化器 + DS²-Instruct（数据合成）= 完整的低成本垂直适配方案

---

### 3. FlexKV 统一 KV Cache 方案
**能做什么：** 如果同时用 vLLM 和 TensorRT-LLM，现在可以用 FlexKV 统一 KV Cache 卸载策略

**落地场景：** 混合部署（vLLM 负责通用模型、TRT-LLM 负责性能敏感模型）时，KV Cache 管理不再分裂

**预期效果：** 运维复杂度下降，prefix cache 命中率提升
